# New2Signal

AI 기반 Discord 뉴스 브리핑 봇입니다.

> 뉴스는 많습니다. 중요한 건 시그널입니다.

## 소개

New2Signal은 RSS 뉴스 데이터를 수집하고, 중복을 정리하고, 분류한 뒤, 하루치 핵심 시장 브리핑으로 정리해서 Discord 안에서 바로 볼 수 있게 해주는 봇입니다.

단순히 뉴스를 모아 보여주는 것이 아니라, 사용자가 빠르게 핵심 흐름을 파악할 수 있도록 저장된 브리핑 형태로 제공합니다. 서버별로 브리핑 채널, 시간, 관심 분야를 설정할 수 있고, 설정된 시간에 자동으로 브리핑을 보낼 수 있습니다.

## 주요 기능

- RSS 뉴스 수집
- 기사 정규화 및 중복 제거
- 규칙 기반 분류 + 선택적 AI 분류
- 일일 시장 브리핑 생성 및 저장
- `/지금브리핑`으로 최신 저장 브리핑 조회
- `/서버설정`으로 서버 설정 확인 및 변경
- `/핑`으로 봇 상태와 지연시간 확인
- 서버별 자동 브리핑 전송
- 감사 로그(action log) 기록

## 명령어 설명

### `/지금브리핑`

최신 저장 브리핑을 현재 채널에 표시합니다.

포함 내용:

- 오늘의 핵심 시장 이슈
- 이슈와 연관된 종목/기업 정보
- 기사 링크
- 한줄 코멘트
- 시장 뉴스 추가 보기 버튼

### `/서버설정`

현재 서버의 브리핑 설정을 보여줍니다.

표시 항목:

- 브리핑 채널
- 브리핑 시간
- 알림 상태
- 관심 분야

버튼:

- `📢 채널 설정`
- `⏰ 시간 변경`
- `🔔 알림 ON/OFF`
- `🧩 관심 분야 설정`

모든 사용자가 볼 수 있지만, 실제 변경은 서버 관리 권한 또는 관리자 권한이 있는 사용자만 할 수 있습니다.

### `/핑`

봇이 정상 동작 중인지 빠르게 확인합니다.

## 동작 구조

간단한 처리 흐름은 다음과 같습니다.

```text
RSS 수집
-> 기사 정리
-> 날짜 필터링
-> 중복 제거
-> 기사 분류
-> 브리핑 생성
-> DB 저장
-> Discord 명령어 / 예약 전송에서 조회
```

핵심은 Discord에서 볼 때 실시간으로 다시 생성하지 않는다는 점입니다.  
브리핑은 먼저 생성되어 DB에 저장되고, 사용자는 저장된 결과를 빠르게 조회합니다.

## 특징

- Discord 안에서 바로 사용하는 브리핑 UX
- 저장 기반 조회라서 빠르고 안정적임
- 서버별 설정 분리 지원
- 버튼, 드롭다운, 모달을 활용한 인터랙션
- 사용자/서버/멤버/설정/감사 로그까지 정규화된 DB 구조
- 권한 없는 사용자는 설정을 볼 수만 있고 변경은 불가

## 사용 안내

### 일반 사용자

- `/지금브리핑` 사용 가능
- `/서버설정` 사용 가능
- 현재 서버 설정 확인 가능
- 설정 버튼은 보이지만 변경 권한은 없음

### 서버 관리자

다음 항목을 변경할 수 있습니다.

- 브리핑 채널
- 브리핑 시간
- 알림 ON/OFF
- 관심 분야

### 서버 설정 권한

설정 변경은 실시간 Discord 권한으로 판단합니다.

- 서버 관리
- 관리자

둘 중 하나가 있어야 변경할 수 있습니다.
