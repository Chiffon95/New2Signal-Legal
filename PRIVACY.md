# Privacy Policy

Last updated: 2026-04-24

## 1. Overview

New2Signal ("the Service") is a Discord bot that collects news from RSS feeds, processes and classifies articles, generates structured market briefings, and delivers those briefings to Discord servers.

This Privacy Policy explains what information is collected, how it is used, how long it is retained, and how users or server administrators can request changes or deletion.

## 2. Information We Collect

New2Signal may collect and store the following information as needed to operate the Service:

### 2.1 Discord Server Information

* Discord server ID
* Discord server name
* Server-specific bot configuration
* Configured delivery channel ID
* Briefing delivery time
* Enabled or disabled notification status
* Selected briefing categories
* Timezone and related server settings

### 2.2 Discord User Information

* Discord user ID
* Username
* Display name
* Guild nickname, where applicable

### 2.3 Guild Membership Information

* Relationships between users and servers in which the bot is used
* Join-related or last-seen related metadata, where applicable for service operation or audit purposes

### 2.4 Audit and Usage Logs

* User actions performed through the bot
* Settings changes made through commands, buttons, selects, or modals
* Action timestamps
* Before/after values for tracked configuration changes

### 2.5 News and Briefing Data

* RSS article metadata and normalized article content as needed for briefing generation
* Classified article data
* Generated daily briefing data
* Article counts and coverage metadata

## 3. How We Use Information

We use collected information only as necessary to provide and improve the Service, including:

* Delivering scheduled briefings to configured Discord channels
* Rendering current stored briefings through slash commands
* Saving server-specific settings
* Supporting interactive configuration features
* Maintaining audit logs for important configuration or interaction events
* Monitoring service reliability, debugging, and abuse prevention
* Improving classification and briefing quality

## 4. Legal and Functional Basis

Data is processed only to the extent necessary to operate the bot's stated features. New2Signal is not intended to collect unnecessary personal information beyond what is required for Discord bot functionality, server configuration, logging, and delivery.

## 5. Third-Party Services

New2Signal may rely on third-party services to operate, including:

* Discord, for bot platform functionality
* RSS feed providers, for news ingestion
* Hosting or infrastructure providers
* Database providers
* LLM or AI API providers, if enabled in deployment configuration

If an LLM provider is used, article content or related briefing-generation input may be transmitted to that provider solely for classification, summarization, or briefing generation as part of the Service.

New2Signal does not sell user data.

## 6. Data Retention

Data is retained only as long as reasonably necessary for operation, reliability, auditing, or troubleshooting.

Typical retention categories may include:

* Server settings: retained while the bot remains installed or until changed/deleted
* User and guild identity references: retained as needed for relational integrity and audit history
* Audit logs: retained for operational and accountability purposes
* Generated briefings and article metadata: retained for service history, browsing, caching, and reliability unless removed

Retention periods may change as the Service evolves.

## 7. Data Sharing

New2Signal does not sell or rent personal data.

Information may be shared only when necessary with service providers used to operate the bot, such as infrastructure, database, logging, or AI service providers, and only to the extent required for service functionality.

## 8. Data Security

Reasonable technical and organizational measures are used to protect stored data. This may include access controls, secret management, restricted operational access, and infrastructure-level security measures.

However, no system can be guaranteed to be completely secure.

## 9. User and Server Administrator Rights

Users or server administrators may request:

* clarification about stored data
* correction of inaccurate configuration-related information
* deletion of server-related configuration where applicable
* deletion requests related to stored data, subject to technical and legal constraints
